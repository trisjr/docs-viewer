# Use Case 02: Tải lên & Xem tài liệu

> Phủ MVP-Scope items 1/2/3/6: upload & xem PDF, `.docx`, `.xlsx` trên một giao diện xem thống nhất.

| Thuộc tính        | Mô tả                                                                                                 |
| :---------------- | :---------------------------------------------------------------------------------------------------- |
| **Actor**         | Người dùng (User) — người mở web app để xem tài liệu.                                                  |
| **Precondition**  | User đang ở giao diện DocsViewer trên một trình duyệt hiện đại; có sẵn file tài liệu cần xem.          |
| **Trigger**       | User chọn tải lên (upload) một file tài liệu.                                                          |
| **Postcondition** | Tài liệu được render và hiển thị trên giao diện xem thống nhất; User có thể điều hướng/zoom cơ bản.    |

## Main Flow

1. User mở giao diện DocsViewer.
2. User chọn file để upload (PDF, `.docx`, hoặc `.xlsx`).
3. Hệ thống kiểm tra định dạng file thuộc 3 loại được hỗ trợ.
4. Hệ thống kiểm tra dung lượng file nằm trong ngưỡng cho phép.
5. Hệ thống nạp và parse nội dung file bằng thư viện render tương ứng (PDF / `.docx` / `.xlsx`).
6. Hệ thống render tài liệu lên giao diện xem thống nhất:
   - PDF: hiển thị theo trang, hỗ trợ phân trang & zoom cơ bản.
   - `.docx`: hiển thị text, heading, bảng, hình cơ bản.
   - `.xlsx`: hiển thị sheet/bảng tính dạng grid (giá trị ô; công thức hiển thị kết quả).
7. User điều hướng trong tài liệu (cuộn trang, đổi sheet, zoom).

## Alternative Flows

- **A1 — Tài liệu nhiều trang/nhiều sheet (PDF, `.xlsx`):** Tại bước 7, hệ thống cung cấp điều khiển phân trang (PDF) hoặc chuyển tab sheet (`.xlsx`); User điều hướng giữa các trang/sheet rồi quay lại Main Flow.
- **A2 — Mở tài liệu mới khi đang xem:** Tại bước 7, User chọn upload file khác; hệ thống thay thế nội dung đang xem bằng tài liệu mới (quay về bước 3).

## Exception Flows

- **E1 — Định dạng không được hỗ trợ:** Tại bước 3, nếu file không thuộc PDF/`.docx`/`.xlsx`, hệ thống từ chối và hiển thị thông báo nêu rõ các định dạng được hỗ trợ. Use case kết thúc. _(PowerPoint `.pptx` và định dạng khác: Out of scope — defer Full Scope.)_
- **E2 — File quá lớn:** Tại bước 4, nếu dung lượng vượt ngưỡng cho phép của trình duyệt, hệ thống từ chối upload và thông báo lý do. Use case kết thúc.
- **E3 — File hỏng hoặc không parse được:** Tại bước 5, nếu nội dung không hợp lệ/hỏng khiến parse thất bại, hệ thống hiển thị thông báo lỗi và không render. Use case kết thúc.
