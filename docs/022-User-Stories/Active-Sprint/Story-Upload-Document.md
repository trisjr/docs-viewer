## Đóng góp cho Sprint Goal

Là **điểm vào (entry point)** của vertical slice: hiện thực luồng `open(file)` để một PDF hợp lệ được validate và khởi tạo `DocumentSession`, bàn giao cho pipeline View/Extract. Không có ST-01, không có gì để "Mở → Xem → Trích xuất".

## Task Breakdown (Engineer thực hiện)

- [ ] Hiện thực `validateUpload()` trong `src/app/validation.ts`: detect format theo MIME/extension + đối chiếu `MAX_FILE_SIZE` (PDF = 25 MB, SDD §8) → trả `UploadValidationResult` (`UNSUPPORTED_FORMAT` / `FILE_TOO_LARGE` / `CORRUPT`).
- [ ] Wire `src/ui/UploadZone.tsx`: file picker + drag-and-drop, gọi validate, hiển thị thông báo lỗi thân thiện (FR-01.3) thay placeholder.
- [ ] Hiện thực phần đầu của `DocumentService.open()` (`src/app/DocumentService.ts`): validate → resolve adapter qua `AdapterRegistry` → khởi tạo `DocumentSession`, dispatch `SESSION_OPENED` (hoặc `ERROR`).
- [ ] Validate **hoàn tất TRƯỚC** khi đẩy tài liệu vào pipeline View/Extract (AC "Validate hoàn tất trước khi bàn giao").
- [ ] Viết test cho `validateUpload` (happy + 3 unhappy: sai format / quá size / file hỏng).

## Definition of Done (story-level — map từ AC canonical)

- [ ] File PDF/`.docx`/`.xlsx` hợp lệ trong ngưỡng → chấp nhận + bàn giao pipeline.
- [ ] Drag-and-drop nhận diện định dạng như chọn file.
- [ ] Sai định dạng → reject + nêu rõ định dạng hỗ trợ (Unhappy Path 1).
- [ ] Vượt `MAX_FILE_SIZE` → reject + nêu rõ giới hạn (Unhappy Path 2).
- [ ] File hỏng → báo lỗi rõ, không render sai (Unhappy Path 3).

> Lưu ý Sprint 1: chỉ **PDF** được wire end-to-end qua adapter thật; `.docx`/`.xlsx` vẫn được validate đúng nhưng view/extract của chúng là Out of Scope sprint này.
