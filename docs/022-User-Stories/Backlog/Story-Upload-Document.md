# User Story: Tải lên tài liệu

**As a** Người dùng DocsViewer,
**I want to** chọn hoặc kéo-thả một file `.docx`/`.xlsx`/PDF để tải lên,
**So that** tôi có thể bắt đầu xem tài liệu trên web mà không cần cài phần mềm.

## Acceptance Criteria

```gherkin
Scenario: Tải lên file đúng định dạng và trong ngưỡng dung lượng
  Given tôi đang ở giao diện DocsViewer
  When tôi chọn (hoặc kéo-thả) một file PDF/.docx/.xlsx hợp lệ trong ngưỡng `MAX_FILE_SIZE` (NFR-07 §4.1)
  Then hệ thống chấp nhận file
  And hệ thống bàn giao tài liệu cho lớp xử lý để khởi tạo phiên xem

Scenario: Hỗ trợ kéo-thả file
  Given tôi đang ở giao diện DocsViewer
  When tôi kéo một file .docx hợp lệ và thả vào vùng upload
  Then hệ thống nhận diện định dạng và chấp nhận file giống thao tác chọn file

Scenario: Validate hoàn tất trước khi bàn giao
  Given tôi vừa chọn một file hợp lệ
  When hệ thống kiểm tra định dạng và dung lượng
  Then việc validate phải hoàn tất thành công TRƯỚC khi tài liệu được đẩy vào pipeline View/Extract
```

## Unhappy Path

```gherkin
Scenario: Định dạng không được hỗ trợ
  Given tôi đang ở giao diện DocsViewer
  When tôi tải lên một file không thuộc PDF/.docx/.xlsx (ví dụ .pptx)
  Then hệ thống từ chối file
  And hiển thị thông báo nêu rõ các định dạng được hỗ trợ

Scenario: File vượt ngưỡng dung lượng
  Given tôi đang ở giao diện DocsViewer
  When tôi tải lên một file vượt ngưỡng `MAX_FILE_SIZE` (NFR-07 §4.1)
  Then hệ thống từ chối upload
  And hiển thị thông báo nêu rõ giới hạn dung lượng

Scenario: File hỏng / không parse được
  Given tôi đã tải lên một file đúng định dạng nhưng nội dung bị hỏng
  When hệ thống cố parse file
  Then hệ thống báo lỗi rõ ràng và không render nội dung sai
```
