# User Story: Xem tài liệu Word (.docx)

**As a** Người dùng DocsViewer,
**I want to** xem nội dung file `.docx` (text, heading, bảng, hình cơ bản) trên web,
**So that** tôi đọc được tài liệu Word ở mức trung thực chấp nhận được mà không cần cài Microsoft Word.

## Acceptance Criteria

```gherkin
Scenario: Render nội dung .docx ở Acceptable Fidelity
  Given tôi đã tải lên một file .docx hợp lệ
  When hệ thống render tài liệu
  Then text, heading, bảng và hình cơ bản được hiển thị ở mức trung thực chấp nhận được (KR1.2)
  And thời gian mở & hiển thị trang đầu của tài liệu kích thước trung bình (baseline NFR-01 §4.2) ≤ 3 giây (KR1.3)

Scenario: Giữ cấu trúc văn bản cơ bản
  Given tôi đang xem một .docx có heading và bảng
  When nội dung được render
  Then phân cấp heading và cấu trúc bảng được giữ theo định nghĩa Acceptable Fidelity (NFR-02)

Scenario: Cuộn xem toàn bộ nội dung
  Given tôi đang xem một .docx dài
  When tôi cuộn trong giao diện xem
  Then toàn bộ nội dung tài liệu truy cập được liên tục
```

## Unhappy Path

```gherkin
Scenario: .docx hỏng / không parse được
  Given tôi tải lên một file .docx có nội dung hỏng
  When hệ thống cố render
  Then hệ thống hiển thị thông báo lỗi và không render nội dung sai

Scenario: Render lỗi một phần (vỡ layout)
  Given một phần nội dung .docx render lỗi nghiêm trọng (mất nội dung/vỡ layout nặng)
  When hệ thống phát hiện lỗi render
  Then hệ thống thông báo trạng thái thay vì hiển thị nội dung sai lệch âm thầm (BR-004-2)
```
