# User Story: Xem tài liệu PDF

**As a** Người dùng DocsViewer,
**I want to** xem tài liệu PDF với phân trang và zoom cơ bản,
**So that** tôi đọc được nội dung PDF trên web mà không cần phần mềm đọc PDF riêng.

## Acceptance Criteria

```gherkin
Scenario: Render PDF và hiển thị trang đầu
  Given tôi đã tải lên một file PDF hợp lệ
  When hệ thống render tài liệu
  Then trang đầu được hiển thị trong giao diện xem
  And thời gian mở & hiển thị trang đầu của tài liệu kích thước trung bình (baseline NFR-01 §4.2) ≤ 3 giây (KR1.3)

Scenario: Phân trang qua nhiều trang
  Given tôi đang xem một PDF nhiều trang
  When tôi điều hướng tới trang kế tiếp / trang trước
  Then hệ thống hiển thị đúng trang tương ứng

Scenario: Zoom cơ bản
  Given tôi đang xem một trang PDF
  When tôi phóng to / thu nhỏ
  Then nội dung trang được zoom tương ứng mà vẫn giữ độ trung thực chấp nhận được (KR1.2)
```

## Unhappy Path

```gherkin
Scenario: PDF hỏng / không parse được
  Given tôi tải lên một file PDF có nội dung hỏng
  When hệ thống cố render
  Then hệ thống hiển thị thông báo lỗi và không render nội dung sai

Scenario: Render lỗi một phần (vỡ layout)
  Given một trang PDF render lỗi nghiêm trọng (mất nội dung/vỡ layout nặng)
  When hệ thống phát hiện lỗi render
  Then hệ thống thông báo trạng thái thay vì hiển thị nội dung sai lệch âm thầm (BR-004-2)
```
