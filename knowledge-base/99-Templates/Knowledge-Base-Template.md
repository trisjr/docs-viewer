# Quy tắc Cấu trúc Hệ thống Tri thức (Knowledge Base Structure Rule)

> Quy tắc này là **BẮT BUỘC** đối với tất cả các tài liệu thuộc **Bộ nhớ dài hạn** (Knowledge Base). Tri thức nền tảng phải được tách biệt rõ ràng với tài liệu dự án cụ thể.

## Các quy tắc nghiêm ngặt (MUST Follow)

1. **Vị trí lưu trữ:** BẮT BUỘC lưu vào thư mục `knowledge-base/`. KHÔNG tạo tri thức nền tảng trong `docs/`.
2. **Hệ thống Dewey:** BẮT BUỘC sử dụng tiền tố **2 chữ số** (01, 10, v.v.).
3. **Phân vùng:** Mỗi file phải nằm đúng phân vùng chức năng (Metas, Technical, Governance, v.v.).
4. **Frontmatter:** BẮT BUỘC có YAML frontmatter với đầy đủ ID, Type, và Status.
5. **Context Hygiene:** Ưu tiên thông tin có tín hiệu cao (high-signal), ngắn gọn, tránh lặp lại nội dung đã có ở file khác.

## Phân biệt KB vs Docs

| Đặc điểm | Knowledge Base (`knowledge-base/`) | Project Docs (`docs/`) |
| :--- | :--- | :--- |
| **Bản chất** | Tri thức nền tảng, tiêu chuẩn, "Bộ nhớ dài hạn". | Tài liệu dự án cụ thể, "Bộ nhớ ngắn hạn". |
| **Ví dụ** | Coding Standards, Quy trình Agile chung. | PRD tính năng X, Sprint 3 Report. |
| **Độ bền** | Lâu dài, ít thay đổi, áp dụng mọi nơi. | Thay đổi theo từng Task/Sprint. |
| **Dewey** | 2 chữ số (VD: `10-Technical`) | 3 chữ số (VD: `030-Specs`) |

## Cấu trúc thư mục Dewey 2 chữ số

```text
knowledge-base/
├── 00-Index.md                # Master Index của toàn bộ tri thức
├── 01-Metas/                  # Từ điển và cấu trúc tư duy
├── 10-Technical/              # Tiêu chuẩn lập trình & hạ tầng
├── 20-Project/                # Quy trình quản trị Agile & Scrum
├── 30-Domain/                 # Tri thức chuyên môn (ví dụ: Fintech, AI, v.v.)
├── 40-Memory/                 # Ký ức tập thể (AAR, Lesson Learned)
├── 45-Role-Memory/            # Ký ức riêng của từng tác nhân / Role
│   ├── software-engineer/
│   ├── context-auditor/
│   └── ...
└── 99-Templates/              # Kho bản mẫu và tiêu chuẩn Prompt
```

## Bản mẫu Frontmatter KB

Mọi tài liệu Knowledge Base BẮT BUỘC phải có:

```yaml
---
id: {KB-CODE}-{NNN}         # Định danh (VD: KB-GLOS-01, KB-SOP-05)
type: knowledge|sop|rule|memory|template
status: live|draft|deprecated
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [tag1, tag2]
linked-to: "./relative-path/Related-KB-Doc.md"
---
```

## Quy tắc Context Engineering (Anthropic)

Khi viết tài liệu cho Knowledge Base, Auditor phải đảm bảo:
1. **Right Altitude:** Trình bày tri thức ở mức độ "vừa đủ" — không quá chi tiết gây bối rối, không quá sơ sài gây hiểu lầm.
2. **Minimal Tokens:** Sử dụng bảng biểu, bullet points thay cho các đoạn văn dài lê thê.
3. **Reference-First:** Nếu tri thức đã được định nghĩa ở file khác, hãy dùng link `[File](./path/File.md)`, đừng copy nội dung.

## Danh sách kiểm tra xác thực

- [ ] File nằm trong `knowledge-base/` và có tiền tố 2 chữ số.
- [ ] Frontmatter đầy đủ các trường bắt buộc.
- [ ] Nội dung đã được tối ưu hóa cho Agent đọc (High-signal).
- [ ] Đã cập nhật `knowledge-base/00-Index.md`.
- [ ] Liên kết chéo (Standard markdown links) được kiểm tra kỹ (không có dead link).
