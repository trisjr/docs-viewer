# 🧠 Hybrid MCP + CLI Strategy — Tối ưu Token cho Agentic Workflows

## 📌 1. Tóm tắt (Summary)

Đúc kết từ task phân tích và cải tiến workflow `/submit-pr`. Khi thiết kế Workflow có sử dụng external tools (MCP servers, CLI), cần áp dụng **chiến lược Hybrid** để cân bằng giữa **Reliability** và **Token Efficiency** thay vì dùng một công cụ duy nhất cho mọi thao tác.

## 🧩 2. Mẫu hình & Giải pháp (Patterns & Solutions)

### A. Core Pattern: Hybrid Tool Selection Matrix

- **Tên mẫu hình:** Tool Selection Matrix cho Agentic Workflows
- **Bối cảnh áp dụng:** Bất kỳ workflow nào phối hợp MCP tools và CLI tools (GitHub, ClickUp, Jira, v.v.)
- **Cách thực hiện:**

  | Đặc tính thao tác | Chọn công cụ | Lý do |
  | :--- | :--- | :--- |
  | Cần truyền/nhận **markdown body dài** (PR body, issue description) | **MCP** | Tránh lỗi shell escaping ký tự đặc biệt |
  | Thao tác **đơn giản, output ngắn** (lấy username, assign, check status) | **CLI** (`gh`, `curl`) | Output gọn → tiết kiệm tokens |
  | **Git thuần** (status, add, commit, push) | **`run_command`** | Tự nhiên nhất, không cần API |
  | Cần **structured JSON response** để chain nhiều bước | **MCP** | Output JSON dễ extract hơn parse text |
  | **Error handling quan trọng** (create/update resources) | **MCP** | Error có cấu trúc, rõ ràng hơn stderr |

### B. Solution Recipe: Cải tiến `/submit-pr`

- **Vấn đề:** Workflow dùng MCP cho mọi GitHub API call → MCP `get_me` trả ~800 tokens output chỉ để lấy 1 username, MCP `issue_write` trả full issue object chỉ để assign.
- **Giải pháp:**
  - `get_me` → `gh api user --jq '.login'` (output ~10 tokens)
  - Auto-Assign → `gh pr edit --add-assignee` (output ~50 tokens)
  - Giữ MCP cho `create_pull_request` và `update_pull_request` (body markdown phức tạp)
- **Kết quả:** Tiết kiệm ~1,240 tokens/lần chạy mà không đánh đổi reliability.

## ⚠️ 3. Bẫy sai lầm & Cách tránh (Pitfalls & Prevention)

- **Lỗi đã gặp:** Dùng `gh` CLI cho PR body chứa markdown phức tạp → lỗi shell escaping (backticks, quotes, newlines bị mất).
- **Nguyên nhân:** Agent tổ hợp chuỗi lệnh shell từ markdown content → ký tự đặc biệt bị interpret bởi shell.
- **Cách phòng ngừa:**
  - **Quy tắc vàng:** Nếu content chứa markdown (heading, code block, checklist) → LUÔN dùng MCP, KHÔNG dùng CLI.
  - Nếu buộc phải dùng CLI cho body dài → ghi ra temp file rồi dùng `--body-file` flag.

- **Lỗi tiềm ẩn:** Dùng MCP cho thao tác đơn giản (chỉ cần 1 field) → lãng phí ~500-800 tokens output mỗi lần.
- **Cách phòng ngừa:** Trước khi chọn tool, tự hỏi: "Output của MCP call này có bao nhiêu % là thông tin tôi thực sự cần?" Nếu < 10% → chuyển sang CLI với `--jq` hoặc `--json` filter.
