## 🧩 2. Mẫu hình & Giải pháp (Patterns & Solutions)

### A. Core Pattern (Mẫu hình cốt lõi)
- **Tên mẫu hình:** **Hybrid Onboarding Context**
- **Bối cảnh áp dụng:** Khi hệ thống có các tài liệu tổng quan lớn phục vụ cả Con người và AI.
- **Cách thực hiện:** 
    1. Tách biệt file **Human-centric** (như README.md) chứa hướng dẫn cài đặt, setup.
    2. Tạo file **Agent-centric** (như AGENTS.md) chứa các nguyên tắc thực thi và lightweight references.
    3. Sử dụng **Just-in-Time Loading** bằng cách chỉ để file path của các Roles thay vì copy toàn bộ mindset matrix vào context.

### B. Solution Recipe (Công thức thành công)

- **Giải pháp:** 
    1. Refactor AGENTS.md thành file context siêu gọn (Minimal High-Signal).
    2. Chuyển các mô tả chi tiết sang các file phụ và chỉ nạp khi Agent cần đóng vai trò đó.
    3. Thêm các chỉ dẫn về **Compaction** và **Structured Note-taking** vào quy trình cốt lõi.
- **Kết quả:** Giảm lượng token nạp ban đầu, dành "Attention Budget" cho các logic phức tạp trong task.

## ⚠️ 3. Bẫy sai lầm & Cách tránh (Pitfalls & Prevention)
- **Lỗi đã gặp:** Để duplicate content giữa README.md và AGENTS.md (như quy trình 6 bước).
- **Nguyên nhân:** Thói quen copy-paste để đảm bảo tài liệu nào cũng đầy đủ, nhưng vô tình làm lãng phí token window của AI.
- **Cách phòng ngừa:** Sử dụng **Single Source of Truth (SSOT)** triệt để. File chính chứa nội dung, file phụ chỉ chứa link tham chiếu kèm mô tả ngắn 1 dòng.
