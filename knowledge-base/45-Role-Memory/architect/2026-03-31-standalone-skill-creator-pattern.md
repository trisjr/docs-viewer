# 🧠 Template: Role Memory Insight: Standalone Skill Pattern & Fallback Mechanism

## 📌 1. Tóm tắt (Summary)
Đúc kết từ quá trình phát triển tự động hóa ClickUp Unit Report. Khi tạo một Skill mới trên TNMCORE-OS để tái phân phối sang nhiều dự án, Skill không thể chỉ là một file hướng dẫn văn bản `SKILL.md` suông. Nó cần được trang bị khả năng Portability (di động) tự cung tự cấp. Tức là nếu dự án chưa thiết lập môi trường (thiếu script thực thi, thiếu template), bản thân file Skill phải chứa sẵn bản dự phòng (fallback) để Agent có thể kích hoạt tại chỗ.

## 🧩 2. Mẫu hình & Giải pháp (Patterns & Solutions)

### A. Core Pattern (Mẫu hình Standalone Skill)
- **Tên mẫu hình:** Standalone Skill Repository (Skill Độc Lập Dự Phòng)
- **Bối cảnh áp dụng:** Khi cần tạo một tác vụ phức tạp đòi hỏi template mẫu hoặc script tự động hóa (Ví dụ: Report Generator, Test Runner) mà User muốn mang sang tái sử dụng (cross-repo) ở dự án khác.
- **Cách thực hiện:** Gói Skill trong 1 thư mục với 3 thành phần:
  1. `SKILL.md`: Chứa logic if-else hướng dẫn Agent ưu tiên file repo trước -> rẽ nhánh gọi file đính kèm skill sau.
  2. Thư mục `scripts/`: Chứa file mã nguồn chạy lệnh.
  3. Thư mục `references/`: Chứa tài liệu, template thiết kế mẫu.

### B. Solution Recipe (Ngữ cảnh ClickUp Report)
- **Vấn đề:** Không phải repo nào cũng có sẵn file `scripts/fetch_all_backlog_tasks.js` và form báo cáo `docs/999-Resources/Templates/Template-Report-Unit.md`.
- **Giải pháp:** Copy hai file này trực tiếp vào `.agent/skills/unit-report-generator/scripts/` và `.agent/skills/unit-report-generator/references/`. Trong `SKILL.md`, dùng lệnh kiểm tra sự tồn tại của file trước khi Agent bắt đầu quá trình đọc raw/template.
- **Kết quả:** Skill trở thành 1 module khép kín đóng gói toàn bộ quy trình từ ETL đến Markdown rendering.

## ⚠️ 3. Bẫy sai lầm & Cách tránh (Pitfalls & Prevention)
- **Lỗi đã gặp:** Hardcode relative path bên trong script. Ví dụ: Dùng `path.join(__dirname, '../../.env')` hoặc lưu file ở cấp bậc cố định từ directory script. Khi chuyển thư mục chạy lệnh hoặc sang project mới sẽ bị báo missing `.env`.
- **Nguyên nhân:** NodeJS chạy theo relative path của file thực thi, không lường trước bối cảnh User chạy lệnh từ project root thay vì trong thư mục.
- **Cách phòng ngừa:** Luôn sử dụng fallback check `process.cwd()` (Current Working Directory). Tìm `.env` ở root của người dùng trước, hoặc xuất kết quả output ra `path.join(process.cwd(), 'resources/...')` để báo cáo luôn sinh ra ở chính thư mục đang thao tác.
